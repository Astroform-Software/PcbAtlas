# Contributing Feedback

Thank you for helping improve PCB Atlas.

PCB Atlas is closed-source commercial software. This repository does not accept product source-code patches, build-system patches, binary submissions, plugin code, or unofficial app distributions. The most valuable contributions here are clear bug reports, safe reproduction material, documentation corrections, and localization feedback.

## Useful Contributions

Please use GitHub issues for:

- Bug reports: crashes, import failures, rendering errors, broken exports, incorrect selection, or confusing app behavior.
- Format reports: real-world PCB format variants, vendor output quirks, import diagnostics, or public sample links.
- Documentation feedback: unclear steps, missing troubleshooting guidance, outdated screenshots, or wrong format notes.
- Localization feedback: mistranslations, awkward wording, untranslated strings, terminology issues, or locale-specific UI problems.
- Feature requests: workflow-level requests that explain the user problem, not implementation patches.

## Not Accepted Here

Please do not submit:

- Source-code patches or pull requests for the closed-source app.
- Repackaged app binaries, modified builds, or reverse-engineered implementation details.
- Private PCB designs, customer data, supplier files, API keys, license data, or commercial pricing material.
- Large confidential project archives without a maintainer-provided private transfer path.

## Bug Report Tips

The best bug report is small, repeatable, and safe to discuss publicly. Include:

- PCB Atlas version and build number.
- Platform, device model, and OS version.
- Input format and import/open path.
- Exact reproduction steps.
- Expected result and actual result.
- Copied error text or Support Bundle text if it is safe to share.
- Redacted screenshots or a short screen recording when useful.

If the issue only reproduces with a private board, describe the symptom and data shape without attaching the file. A maintainer can decide whether a private transfer path is needed.

## Localization Feedback

Localization reports are welcome. Please include:

- Language or locale.
- The exact current text.
- Where it appears in the app or documentation.
- Suggested replacement text.
- Why the replacement is clearer, more accurate, or more natural.

For PCB terminology, prefer wording that helps engineers and manufacturing users understand the app quickly. If a term has multiple local conventions, mention the region or toolchain where your suggestion is common.

## Maintainer Edits

Maintainers may retitle issues, add labels, request a smaller reproduction, move localization notes into a tracking issue, close duplicates, or close reports that cannot be safely investigated with the information provided.
